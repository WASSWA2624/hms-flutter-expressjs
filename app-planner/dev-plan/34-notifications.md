# 34 - Notifications and Communications

## Goal
Manage in-app notifications, delivery state, unread indicators, conversations, messages, workflow reminders, alerts, and user-facing notification badges without creating noise.

## Source of Truth
- Use `app-write-up.md` for notifications and communications scope.
- Use `opd-flow.md` and `ipd-flow.md` for patient-flow notifications such as triage ready, doctor review, lab/radiology results, pharmacy ready, admission, transfer, discharge, billing, housekeeping, and bed readiness.
- Use `01-policy.md` for simple UI, role access, partial badge refresh, and responsive behavior.

## Backend Routes To Align With
- `/api/v1/notifications`
- `/api/v1/notification-deliveries`
- `/api/v1/conversations`
- `/api/v1/messages`
- `/api/v1/communications-workspace`

## Implementation Scope
1. Notification badge and dropdown/panel in the app shell.
2. Notifications workspace with unread, assigned to me, workflow alerts, patient flow, system/admin, and archived filters.
3. Conversation/message views where backend supports communication.
4. Modal actions for mark read/unread, archive, mute/dismiss where supported, open related record, and send message where permitted.
5. Role-aware notifications generated by OPD/IPD/lab/radiology/pharmacy/billing/discharge/housekeeping workflows.

## UI and Workflow Contract
| Area | Requirement |
| --- | --- |
| Badge | Show clear unread count and update it without reloading the app. |
| Notification list | Keep titles short, actionable, and staff-friendly. Show patient/context only where permitted. |
| Actions | Use modals or lightweight panels for quick actions; deep-link only when opening the related record is needed. |
| Noise control | Avoid duplicate alerts for the same event/user. Group where backend supports it. |
| Responsiveness | Mobile uses notification page/panel; desktop may use app-bar dropdown plus full workspace. |

## Flow Synchronization Rules
- OPD/IPD workflow updates should notify only responsible roles/users.
- Result-ready notifications must open the correct clinical review context.
- Billing, pharmacy, discharge, bed, and housekeeping notifications must link to the correct queue item.
- Mark-read/archive must update badge and list item only.

## Access and State Rules
- Users see only notifications allowed by role, tenant, facility, department/unit, patient access, and module entitlement.
- Notification content should avoid exposing sensitive details to unauthorized roles.
- After mutation, refresh only the notification item, badge count, related queue row, and opened detail link if affected.

## Reports and Printing
Notification delivery summaries, message audit extracts, and communication logs must use generated report templates from `35-reports-audit.md` where exports are permitted.

## Done Criteria
- Users can quickly find and act on relevant alerts.
- Notification badge and counts update predictably.
- Workflow notifications link to correct OPD/IPD/module records.
- No noisy, duplicate, or unauthorized notifications are shown.
